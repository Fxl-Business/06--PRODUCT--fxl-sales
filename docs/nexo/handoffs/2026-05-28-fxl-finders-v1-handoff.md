# Handoff: FXL Finders v1.0
> Generated by /nexo:add-feature on 2026-05-28
> Milestone: v1.0 — FXL Finders MVP | Phases: 01–06

## TL;DR

FXL Finders is a new standalone B2B affiliate SaaS, built on the FXL canonical template, that extracts the finders program out of fxl-financiero's admin panel into a multi-product platform any FXL SaaS can plug into. Independent Finders sign up (public form → admin approval → Clerk invite), generate signed referral links for a chosen product at a self-set price within an admin-configured `(min, list, max)` band, and earn commission when the referred sale closes. Sibling apps report sales via an HMAC-signed, idempotent webhook; commissions move `pending → locked → paid` (30-day hold) and are paid out via manual CSV export. In-house Sellers are first-class (co-attribution). v1.0 ships the platform **plus** the live fxl-financiero integration (cross-repo, as a reviewable patch). All money/tenant tables are protected by Postgres FORCE RLS keyed on `org_id`; every money mutation writes a hash-chained audit row.

## Known soft spots

Double-check before shipping (from SUMMARY deviations + the integration audit):

- **conversions `customer_email` is `z.string().email()` (strict)** — a malformed/empty email from financiero → 400, and the sender retries only on 5xx, so that sale silently fails to book. Decide validation at the financiero boundary or relax the schema before wiring real data. _(integration audit W1)_
- **`external_order_id = orgId`** in the patch (spec illustrated `subscription_id`) — fine for one-sale-per-org; a re-subscribe-after-churn second sale would dedupe-drop. _(W2)_
- **`finders.clerk_user_id`/`clerk_org_id` + `sellers.clerk_user_id` are NULLABLE** (not `''`) — a deliberate fix vs the plan's `''` placeholder, which collided with the UNIQUE constraint.
- **audit_log pre-Phase-05 rows** (Phases 02/03 admin writes) used `''` hash placeholders and sit outside the hash chain; new rows chain from genesis.
- **Phase 06 husky bypass (cross-repo):** the local `feat/fxl-finders-integration` commit in the fxl-financiero repo was made with the husky hook bypassed (its hook would re-dirty that repo's tracked docs). The canonical deliverable is the reviewable **patch**; the local branch is a convenience. Review the patch, don't merge the branch blindly.
- **vitest is v2.1.9** in apps/api (no `test.projects`) — integration tests use a `VITEST_INTEGRATION` env-flag split + `fileParallelism:false` (shared test DB).

## Assumptions made

Decided during planning/execution (sources: spec § 2 decision table + plan-brief D-A..D-R + Wave outcomes):

- Scope = platform + fxl-financiero only; onboarding = public signup + admin approval; payout = manual CSV; price band = per-product (min,list,max); sellers first-class w/ Clerk login.
- Commission rates per-product flat 30/20/12 (placeholder); attribution + hold both 30 days; sale-close trigger = financiero `first_paid_at`.
- Single canonical app slug `fxl-financiero`; one `payouts` table (no batch tables); HMAC payload `ts + "." + rawBody`; idempotency_key `sha256(source+external_order_id+event_type)`.
